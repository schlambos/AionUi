# AionUi v3 交互设计概念文档

> 配合 `v3-layout.html` wireframe 阅读。每次讨论调整交互后同步更新本文档。
>
> **v3 核心变更**：侧边栏从 2-Tab 切换重构为 **Icon Rail + 列表面板** 结构，每个功能模块拥有独立入口。

---

## 产品定位

- **短期**：多 CLI Agent 聚合工具（基于 ACP 协议）
- **长期**：人格化助手平台，所有 agent（含 CLI）统一为"助手"概念，拥有名字、头像、身份

## 核心概念

### AionCLI（秘书）

- AionCLI 是用户的**专属秘书**，始终排在通讯录助手列表第一位
- AionCLI 落地页 = App 默认视图，落地页发起的会话归属于 AionCLI
- AionCLI 在通讯录中有「秘书」标签，浅绿色背景高亮，与普通助手视觉区分
- 秘书是一个真正的助手，有自己的会话历史和落地页

### 助手（Assistant）

- 所有 agent 统称"助手"，不区分来源（内置预设 / CLI / 市场安装）
- CLI agent 只是一种"没有自定义名字和头像"的助手，使用 CLI 的名称和 logo 作为默认身份
- 内置预设助手有人像头像（JPG 肖像），CLI 助手使用产品 logo（SVG/PNG）
- 助手是用户主动创建或添加的**正式角色**，常驻通讯录

### 临时队员（Temporary Member）

- 团队任务执行过程中，Leader 可根据任务需要**动态生成临时队员**（如"前端开发"、"测试工程师"、"文档编写"）
- 临时队员有名字，会被指定 LLM 和 Agent，**头像沿用底层 Agent 的头像**，但不算正式助手，不出现在通讯录"助手"中
- 临时队员在任务结束后**不会自动消失**，会保留在团队成员中
- **保存为助手**：
  - 用户可在团队成员列表中手动将临时队员保存为正式助手，保存后升级为常驻助手
  - Leader 也会在群聊中主动建议：当临时队员表现良好时，发送一个角色卡片（含「💾 保存为助手」按钮），引导用户保存
- 视觉区分：
  - 头像使用底层 CLI Agent 的 logo，外框为**虚线边框**（区别于正式助手的实线/无框）
  - 名字后缀**（临时）**，如「前端开发（临时）」
- 成员列表中的角色标签：Leader 显示「Leader」，队员显示「队员」，临时队员显示「临时队员」（样式统一，不加斜体）
- 名字中的后缀保持简短**（临时）**，如「前端开发（临时）」，角色标签才显示完整的「临时队员」

### 团队（Team）

- 本质是"一组助手 + 一种工作方式"的容器，人员和工作流固定
- 一个团队可以有多个会话，每个会话可绑定不同工作空间
- 团队中的每个会话代表一个独立任务

### 会话（Conversation）

- 助手和团队都是"对话容器"，各自可拥有多个会话
- 每个会话可选绑定一个工作空间（项目文件夹）

### 工作空间（Workspace）

- 对应一个本地项目文件夹
- 会话的可选属性，不是必须

## Icon Rail + 列表面板

侧边栏由两部分组成：

### Icon Rail（窄图标栏，56px）

左侧全高窄栏，包含应用级导航。每个按钮由**图标 + 文字标签**组成（类似 Slack 侧边栏），通讯录排在第一位（平台核心是"找人对话"）：

```
[Logo]
───
👥 通讯录        ← 默认选中，第一入口
💬 工作区        ← 未读数 Badge
⏰ 定时          ← 有活跃任务时绿点
───
⚙️ 设置
👤 用户头像      ← 点击弹出用户菜单
```

- 每个按钮包含 16px 图标 + 9px 文字标签，纵向排列
- 选中的图标有绿色高亮背景
- 未读消息聚合在工作区图标上显示红色 Badge
- 定时任务有活跃任务时显示绿色小圆点
- Logo 点击回到 AionCLI 落地页

### 列表面板（256px）

Rail 右侧的面板区域，根据选中的 Rail 图标切换内容：

| Rail 图标 | 面板标题 | 面板内容 |
|----------|---------|---------|
| 👥 通讯录 | 通讯录 | ✏️ 新会话（固定行）+ 🔍搜索 + 助手列表 + 团队列表 |
| 💬 工作区 | 工作区 | ✏️ 新会话（固定行）+ 🔍搜索 + 工作区分组会话列表 |
| ⏰ 定时 | 定时任务 | ＋新任务 + 🔍搜索 + 活跃/暂停任务列表 |
| ⚙️ 设置 | 设置 | AI 配置（Gemini 设置、模型平台、助手管理、Agent 设置、功能扩展）/ 应用（显示与主题、手机/网页访问、桌面宠物）/ 其他（系统设置、关于），点击项目在右侧打开设置详情页 |

### 「新会话」固定行

- 每个面板（通讯录、工作区）顶部有一个**固定的「✏️ 新会话」行**
- 点击「新会话」→ 选中 AionCLI，显示其落地页，取消所有会话选中
- 刷新 / 首次进入 App 时，默认选中 AionCLI，显示其落地页
- 搜索栏紧跟在「新会话」下方，圆角胶囊样式

### Tab 切换记忆

- 每个 Tab 记住用户上次选中的会话
- 切换 Tab 时自动恢复上次选中的会话和对应的右侧视图
- 点击「新会话」时不影响其他 Tab 的记忆

### 💬 聊天面板

- 工作区文件夹展开后显示该工作区下的所有会话（单聊 + 团队会话）
- "会话" 区域收纳未绑定工作空间的会话
- **默认状态**：有未读消息的文件夹自动展开，无未读的默认收起

### 👥 通讯录面板

- **助手**：
  - **AionCLI 始终排第一位**，带「秘书」标签，浅绿背景，落地页的会话归属于此
  - 标题栏右侧有 ＋ 按钮，点击打开创建助手向导
  - 内置预设助手默认预装，首次即可见
  - CLI agent 检测到后自动添加
  - 助手一旦添加就常驻列表，无论是否有过会话
- **团队**：
  - 标题栏右侧有 ＋ 按钮，点击打开团队成员管理弹窗（创建新团队）
  - 仅在用户主动创建团队后出现
  - 团队描述显示「Leader名 等 N 位成员」格式（如「Cowork 等 4 位成员」），类似微信群的成员概要
- **默认状态**：所有助手和团队的会话列表默认收起

### ⏰ 定时任务面板

- 分为"活跃任务"和"已暂停"两个区域
- 每个任务显示：关联助手头像 + 任务名称 + 执行频率（如"每天 09:00"）+ 上次执行状态与时间
- 执行状态：`✓ 上次成功`（绿色）或 `✕ 上次失败`（红色）+ 时间戳
- 点击任务进入详情编辑页

### ⚙️ 设置面板

- 分为三组：AI 配置（Gemini 设置、模型平台、助手管理、Agent 设置、功能扩展）、应用（显示与主题、手机/网页访问、桌面宠物）、其他（系统设置、关于）
- 点击设置项在右侧内容区打开对应设置页（详见「设置」章节）

## 未读消息与排序

### Badge 语义

- 所有 badge（气泡数字）统一表示**未读消息数**，不表示会话总数
- 会话总数无需提醒，展开即可见

### 视觉规范

- **Badge 颜色统一红色**（`#f44336`），包括：侧边栏文件夹/助手/团队的聚合数字
- **未读会话标识**：加粗文字（`font-weight: 600`），不使用红色圆点

### 层级提醒规则

| 层级 | 提醒方式 |
|---|---|
| 会话条目 | 有新消息时**加粗会话名**（正常字重 = 已读） |
| 父节点（助手/团队/文件夹） | 聚合显示**未读消息总数**红色 badge |

### 会话行设计（两个面板统一规则）

```
会话名称          [人/团队头像] [📁 工作空间]
```

**未读标识：**
- 已读 = 正常字重（默认状态）
- 未读 = **加粗文字**（`font-weight: 600`），不使用红色圆点（避免视觉上像 bug）

**行尾标记（左 → 右，可叠加）：**

| 位置 | 内容 | 说明 |
|---|---|---|
| 人/团队头像 | 助手小头像（圆形）或团队叠加头像（两圆横向叠放） | 聊天 Tab 显示，通讯录 Tab 不显示（父级已标明） |
| 📁 工作空间 | 工作空间名称（max-width 截断） | 通讯录 Tab 显示，聊天 Tab 不显示（已有文件夹分组） |

**不显示的内容：**
- 不显示时间戳
- 不显示 ⏰ 定时任务标记（定时任务管理有独立入口）

### 通讯录 Tab 排序

- 按最新消息时间倒序排列
- 有未读消息的助手/团队排在最上方

### 通用排序规则

- **所有层级**都按最新消息时间倒序排列
  - 父节点之间：有新消息的排最上面
  - 同一父节点下的会话：有新消息的会话冒到最上面
- 文件夹/助手/团队的 badge 为其下所有会话的未读数之和

## Hover 操作

### 通讯录 Tab

**助手条目 hover：**
- ··· 更多 →
  - ＋ 新建会话
  - 📌 置顶 / 取消置顶（置顶后出现在通讯录顶部"置顶"分区，助手和团队混排）
  - 查看详情
  - 🗑 移除助手（⚠️ 二次确认：提醒助手下的所有会话将被删除）

**团队条目 hover：**
- ··· 更多 →
  - ＋ 新建会话
  - 📌 置顶 / 取消置顶
  - 团队设置
  - 🗑 移除团队（⚠️ 二次确认：提醒团队下的所有会话将被删除）

**通讯录置顶分区：**
- 有任何助手或团队被置顶时，通讯录 Tab 顶部出现"置顶"分区
- 助手和团队混排，按置顶时间排序

### 聊天 Tab

**会话条目 hover：**
- ··· 更多 →
  - 📌 置顶 / 取消置顶（置顶后出现在聊天 Tab 顶部"置顶"分区）
  - ✏️ 重命名
  - 🗑 删除会话

**聊天置顶分区：**
- 有任何会话被置顶时，聊天 Tab 顶部出现"置顶"分区

**工作区文件夹 hover：**
- ··· 更多 →
  - ＋ 新建会话 → 弹出选择"助手"或"团队"（新会话默认名称"新会话"，后续用户可改）
  - 📁 在 Finder 中打开
  - 🗑 移除工作区（⚠️ 二次确认：提醒工作区下的所有会话将被删除）

## 默认视图（AionCLI 落地页）

### 概念

- **没有独立首页**——App 启动时默认选中通讯录中的 AionCLI，显示其落地页
- AionCLI 是万能入口 / 总调度——用户不知道找谁时，直接跟秘书说就行
- 每个助手和团队在通讯录中点击后都有自己的落地页，AionCLI 的落地页只是其中之一
- 落地页的会话列表点击后自动跳转到工作区面板对应会话

### 落地页布局（助手通用）

```
┌────────────────────────────────────────────┐
│           [助手头像 72px 圆形]               │
│         助手名称                             │
│    助手描述文字                               │
│         [ 输入框 + 发送按钮 ]                │
│  [示例提示词1] [示例提示词2] [示例提示词3]       │
├────────────────────────────────────────────┤
│  最近会话                                    │
│  **重构侧边栏组件**    📁 AionUi             │
│  修复登录 bug         📁 AionUi             │
│  帮我做个PPT                                │
│                                            │
│  ＋ 创建新助手 | ⏰ 定时任务 | ⚙️ 设置            │
└────────────────────────────────────────────┘
```

- **助手区**：头像（AionCLI 带绿色边框光晕）+ 名称 + 描述 + 输入框 + 示例提示词胶囊
- **最近会话**：该助手/团队下的会话列表
  - 有未读消息的会话用**加粗文字**标识（不使用红色圆点，避免视觉上像 bug）
  - 每行显示会话名 + 工作空间标签（如有）
  - 点击会话 → 自动切换到工作区面板，展开对应文件夹，选中该会话
- **快捷入口**：底部一行灰色文字链接，hover 变绿

### 默认入口

| 入口 | 行为 |
|---|---|
| App 启动 | 选中通讯录 AionCLI，显示其落地页 |
| ＋ 新会话 | 选中通讯录 AionCLI，显示其落地页 |
| 点击 Logo | 选中通讯录 AionCLI，显示其落地页 |

三个入口行为一致：切到通讯录面板 + 选中 AionCLI + 显示落地页。

## 整体布局

### 结构

页面分为三个区域，没有全局顶部标题栏：

| 区域 | 宽度 | 说明 |
|---|---|---|
| **Icon Rail** | 56px | 左侧窄栏，全高拉通。Logo + 模块导航图标 + 设置 + 用户头像 |
| **列表面板** | 256px | Rail 右侧，全高拉通。面板标题 + 前进后退 + 对应模块的列表内容 |
| **右侧内容区** | 剩余空间 | 全高拉通。根据状态显示落地页 / 单聊 / 团队会话 / 设置详情 |

### Sidebar Wrapper 结构

侧边栏整体由 `.sidebar-wrapper` 包裹，顶部是全局工具栏（红绿灯 + ◫←→），下方是 Rail + 面板：

```
┌─ sidebar-wrapper ──────────────────────┐
│ [🔴🟡🟢]                    [◫] [←] [→] │
├── sidebar-body ────────────────────────┤
│ [Rail]  │  [列表面板]                    │
└────────────────────────────────────────┘
```

### Icon Rail

```
  [Logo]
  ──
  [👥] 通讯录   ← 默认选中
  [💬] 工作区   ← badge
  [⏰] 定时     ← 绿点
  ──
  [⚙️] 设置
  [👤]
```

- Logo 点击回到 AionCLI 落地页
- 每个按钮包含图标 + 文字标签，纵向排列（类似 Slack）
- 导航图标之间有分隔线区分主功能和系统功能
- 用户头像点击弹出用户菜单（用量统计、帮助与反馈）
- 远程连接（📱）不再有独立图标，归入设置面板 → "WebUI 远程访问"

### 列表面板顶部

```
面板标题
```

- 标题随 Rail 选中项变化：通讯录 / 工作区 / 定时任务 / 设置
- 前进后退按钮已上移至 sidebar-wrapper 顶部工具栏

## 会话页面

### 通用 Header 结构

单聊和团队会话共用同一种 header 布局：

```
会话标题 🔍              [头像区域] [＋] [◫ 工作空间切换]
```

| 元素 | 说明 |
|---|---|
| 会话标题 | 主标题，如「重构侧边栏组件」「v2.0 侧边栏重设计」 |
| 🔍 | 搜索聊天记录（标题旁小图标） |
| 头像区域 | 单聊 = 1 个头像；团队 = 多个头像叠加 + "N 位成员" 文字 |
| ＋ | 单聊：邀请助手创建团队；团队：合并到成员面板中 |
| ◫ | 展开/收起右侧工作空间面板 |

**不放"更多"操作按钮**——重命名、置顶、删除等已在侧边栏会话列表的 hover 操作中提供。

### 聊天 / 任务 Tab 切换

单聊和团队会话共用同一种 Tab 切换机制，Header 下方有 Tab 栏：

```
💬 聊天  |  📋 任务
```

- **聊天**（默认）：显示消息流 + 底部输入框
- **任务**：显示看板面板（Backlog → Todo → In Progress → Done），隐藏消息和输入框
- Tab 栏贴合 Header 底部，绿色下划线指示当前选中

### 任务看板（Task Board）

```
┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
│ Backlog  │ │  Todo    │ │In Progress│ │  Done    │
│ ● 3   ＋ │ │ ● 2   ＋ │ │ ● 1   ＋  │ │ ● 1   ＋ │
├──────────┤ ├──────────┤ ├──────────┤ ├──────────┤
│ TASK-1   │ │ TASK-4   │ │ TASK-6   │ │ TASK-7   │
│ 任务标题  │ │ 任务标题  │ │ 任务标题  │ │ 任务标题  │
│ 描述...   │ │ 描述...   │ │ 描述...   │ │ 描述...   │
│ [Low][👤]│ │[Med][👤] │ │[High][👤]│ │ [Low][👤]│
└──────────┘ └──────────┘ └──────────┘ └──────────┘
```

- 4 列看板：Backlog（灰）、Todo（蓝）、In Progress（橙）、Done（绿），每列标题旁有彩色圆点 + 计数 + ＋ 按钮
- 任务卡片：ID + 标题 + 描述 + 优先级标签（Low/Medium/High）+ **负责人头像**（20px 圆形，右下角）
- 单聊看板：负责人为该助手头像
- 团队看板：负责人为实际分配的团队成员头像
- 卡片圆角 10px，hover 阴影加深

### 单聊页面

### 团队会话页面

- Header 右侧：多个成员头像横向叠加 + "N 位成员" 文字，整体区域有 hover 背景变化暗示可交互
  - **点击头像区域**：弹出成员简要面板（成员列表 + 邀请成员 + 管理链接）
    - 每位成员显示头像 + 名字 + 角色标签（Leader / 队员 / 临时队员）
    - Leader 标签绿色高亮，队员和临时队员标签均为默认灰色
    - 临时队员的头像使用虚线边框，右侧显示 💾 保存按钮（点击可保存为正式助手）
  - **点击"管理"**：打开「团队成员管理弹窗」
- **消息区**（主群聊 = 会议室，只展示人与人之间的沟通）：

| 场景 | 主群聊展示 |
|---|---|
| 成员加入团队 | 系统消息：居中横线 + 头像 + 「**XXX** 加入团队（由 YYY 邀请）」 |
| Leader 创建临时队员 | 系统消息：居中横线 + 虚线头像 + 「**XXX（临时）** 加入团队（由 YYY 创建的临时队员）」 |
| 用户直接发消息（不@） | 默认发送给 Leader，Leader 自动回复。输入框 placeholder：「发消息给 Leader名，或 @某人...」 |
| 用户 @某人 | 用户消息 + 被@的人回复 |
| Leader → 成员 / 成员 → 成员 / 成员 → Leader | 所有消息都带 `@收件人`，格式：`@xxx，消息内容` |
| 成员自己工作（工具调度、思考、步骤执行） | **不展示**，仅在该成员的独白中可见 |

> **系统消息样式**：`——— [头像] 文字 ———`，居中显示，12px 灰色文字，临时队员头像用虚线边框

> **首次使用提示**：首次进入团队会话时，Header 下方显示浅绿色提示条「💡 开启「只看我的消息」，可过滤成员间通信，只看与你相关的对话。」，可关闭。

> **核心原则 — @ 即投递**：
> - **主群聊 = 共享信箱**，所有出现在此处的消息都有明确的 @收件人（包括成员 @用户时写 `@VeryLiu`），唯一例外是用户不@时 Leader 自动接收
> - **独白 = 个人工作台**，两个人互相发消息必须带 @（`@收件人`），只有该成员自己的独白（思考、执行步骤、自言自语）不需要 @
> - 成员自己的工作过程（工具调度、思考、步骤）不进入主群聊，只在独白中可见

- **「只看我的消息」开关**：Header 中提供 toggle 开关
  - **关闭（默认）**：显示所有消息，包括成员之间的 @通信
  - **开启**：过滤成员之间的内部通信，仅显示与用户相关的消息（用户发送 + 用户收到的回复）
  - 帮助用户在"全局掌控"和"只看结果"两种模式间切换
- **点击成员头像** → 进入该成员的「独白」视图（个人工作台）
- 底部输入框：与单聊一致

### 成员独白（个人工作台）

- **入口**：在团队会话消息区中，点击任意成员头像
- **Header**：标题「XXX 的工作台」+ ← 返回按钮（回到主群聊）
- **内容**：以该成员为视角的完整单聊画面，包含：
  - **用户 @此人的消息**（用户直接发给他的指令/对话，原样展示）
  - 别人发给他的消息（Leader 指令、其他成员协作消息）
  - 他给别人的回复
  - 他自己的工具调度、思考过程、执行步骤
- **消息方向**：用户消息显示在右侧（绿色气泡），其他所有人的消息（包括该成员自己、Leader、其他队员）都显示在左侧（白色气泡）
- 独白是该成员进程的**完整上下文**，主群聊只展示人与人之间的沟通摘要
- 临时队员的独白与正式队员一致，头像使用虚线边框

### 工作空间面板

- 右侧独立面板，全高拉通
- 通过 Header 中的 ◫ 按钮展开/收起
- 内容：
  - 标题：「工作空间」
  - Tab 切换：文件 / 变更 / 终端
  - 搜索框
  - 文件树（目录可展开/折叠，变更文件用绿色标记 M）

## 创建助手（多步向导）

### 概念

- 创建助手 = 为新员工建立入职档案
- 入口：通讯录 Tab → 助手区域 → ＋ 按钮
- 弹窗打开后先展示**选择创建方式**页面，两种入口：
  1. **手动创建**：进入 4 步向导，逐步配置
  2. **与 Agent 共建**：关闭弹窗，跳转新会话，输入框预填「我要创建一个助手，我的需求是...」，通过对话让 Agent 帮助生成配置
- 手动创建采用 **4 步向导**，每步专注一个主题，避免单页过长

### 步骤总览

| 步骤 | 标题 | 对应字段（AcpBackendConfig） |
|---|---|---|
| 第 1 步 | 身份与引擎 | `name`*、`avatar`（emoji 选择+上传）、`description`、`presetAgentType`（Chip 单选）、权限（Chip 单选）、`models`（默认模型下拉） |
| 第 2 步 | 规则与提示词 | `context`（Markdown 系统提示词）、`prompts`（示例提示词列表） |
| 第 3 步 | 技能 | `enabledSkills`（内置技能 toggle）、`customSkillNames`（导入自定义技能） |
| 第 4 步 | 工具 | 工具分类卡片（带 toggle 开关），最后一步按钮为「完成并创建」 |

自动填充（无需用户输入）：`id: custom-${Date.now()}`、`enabled: true`、`isPreset: true`、`isBuiltin: false`

### 布局规则

- **Header**：左侧图标 + 步骤标题 + 面包屑（`第 N 步 > 标题`），右侧 ✕ 关闭
- **Body**：每步独立面板，切换时淡入
- **Footer**：左侧「上一步」（第 1 步隐藏），右侧「下一步：XXX >」（最后一步为「完成并创建」）
- **进度条**：第 2~4 步顶部显示进度条 + 百分比

### 第 1 步：身份与引擎

```
┌─────────────────────────────────────────────────────────┐
│  [✦] 身份与引擎                                      ✕  │
│       第 1 步 > 身份与引擎                                │
├─────────────────────────────────────┬───────────────────┤
│  名称 *  [______________________]  │  ● 助手预览        │
│                                     │  预览助手效果      │
│  头像            ⇧ 上传自定义图片    │  ┌─────────────┐ │
│  [🤖][🧑‍💻][👩‍🔬][🧙][👨‍🎨][🦊][More]  │  │  [🤖]        │ │
│                                     │  │  新建助手     │ │
│  描述                               │  │  AionCLI     │ │
│  [______________________________]  │  │  "我是..."    │ │
│                                     │  │  权限：默认   │ │
│  执行引擎 (presetAgentType)          │  └─────────────┘ │
│  (AionCLI) [Claude Code] [Codex].. │                   │
│                                     │  第 1/4 步        │
│  权限                               │                   │
│  (默认) [自动编辑] [全自动]           │                   │
│                                     │                   │
│  默认模型                            │                   │
│  [跟随引擎默认                  ▾]   │                   │
├─────────────────────────────────────┴───────────────────┤
│  [隐藏]                    [下一步：规则与提示词 >]       │
└─────────────────────────────────────────────────────────┘
```

- 左右分栏：左侧表单，右侧实时预览卡片
- 头像选 emoji 或上传自定义图片
- 执行引擎和权限均为 Chip 单选

### 第 2 步：规则与提示词

- **规则**（`context`）：Markdown 多行编辑器，定义助手行为规则和系统提示词
- **示例提示词**（`prompts`）：可增减的提示词列表，帮助用户快速开始对话

### 第 3 步：技能

- **内置技能**：卡片网格（2 列），每张含图标 + 名称 + 描述 + toggle 开关
- **自定义技能**：已导入列表（初始可为空），底部「导入技能」按钮
- 对应字段：`enabledSkills`、`customSkillNames`

### 第 4 步：工具

- 工具按分类展示为卡片网格（2 列），每张卡片含：图标 + 名称 + 工具列表 + toggle 开关
- 分类：文件系统、网络与浏览、代码与终端、图像与媒体、记忆与规划、助手协作
- 最终按钮：「完成并创建」

### 设置页面布局

```
设置项标题
───────
分区标题
┌──────────────────────────────────────┐
│ 设置行：标签 + 描述         [控件]    │
│ 设置行：标签 + 描述         [控件]    │
└──────────────────────────────────────┘
```

- 每个设置页按分区组织，每个分区有标题 + 白色卡片
- 卡片内为设置行列表，每行：左侧标签+描述，右侧控件（toggle、下拉、按钮等）
- 控件类型：toggle 开关、select 下拉、按钮、文字值+修改按钮、色环选择器
